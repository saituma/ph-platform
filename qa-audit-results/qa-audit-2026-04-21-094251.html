ISS-002: Basic Form Validation in Login

Login and Registration forms rely primarily on native HTML5 validation ('required' attribute). While functional, it lacks the detailed error feedback expected in a premium application.

Expected

Detailed inline validation messages for email format and password requirements.

Actual

ISS-003: Hardcoded API URL fallback

Several components have hardcoded fallbacks to 'http://localhost:3000' if environment variables are missing. This could cause confusion in production environments if not properly configured.

Expected

Fail loudly or use a central configuration service for API URLs.
